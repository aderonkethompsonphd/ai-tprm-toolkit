# Vendor Offboarding Checklist
**AI-TPRM Toolkit | GDPR Article 17 / ISO 27001 Aligned | Version 1.0 | CC BY 4.0 | A. F. Thompson**

---

## Trigger
- [ ] Contract expiry notification received
- [ ] Termination for cause decision made
- [ ] Business decision to change provider
- [ ] Trigger date: ___________ | Target offboarding completion: ___________

---

## Immediate Actions (Within 24 hours of offboarding decision)

- [ ] Vendor owner notified of offboarding
- [ ] IT Security notified to begin access revocation preparation
- [ ] Legal notified to review termination provisions
- [ ] Data Protection Officer notified (if personal data involved)

---

## Access Revocation (Within agreed timeframe — default: 30 days)

- [ ] All logical access to systems revoked (usernames, passwords, API keys, tokens)
- [ ] VPN / remote access credentials revoked
- [ ] Physical access credentials revoked (building passes, key cards)
- [ ] Third-party portal/system access revoked
- [ ] Shared credentials changed
- [ ] Service accounts disabled
- [ ] Access revocation verified by IT Security
- [ ] Access revocation date confirmed: ___________

---

## Data Deletion and Return (GDPR Article 17 / Contract obligation)

- [ ] DPA data deletion/return clause reviewed
- [ ] Data deletion request sent to vendor with deadline
- [ ] Deletion to include all sub-processors (confirm with vendor)
- [ ] Data deletion confirmation received from vendor in writing
- [ ] Data deletion confirmation includes: date, scope, sub-processors covered
- [ ] Deletion confirmation uploaded to vendor record
- [ ] If data returned: receipt confirmed and data transferred to approved storage

---

## Contractual Closure

- [ ] Termination notice served per contract terms
- [ ] Outstanding invoices settled
- [ ] Service credit claims resolved
- [ ] NDA / confidentiality obligations confirmed to survive termination
- [ ] IP ownership / licence termination confirmed
- [ ] Contract termination documentation filed

---

## Knowledge Transfer

- [ ] Documentation received from vendor?
- [ ] System configurations / runbooks received?
- [ ] Transition support period agreed and completed?
- [ ] Replacement vendor / in-house capability confirmed operational?

---

## Post-Offboarding Record

- [ ] Vendor status updated to INACTIVE in TPRM system
- [ ] Offboarding record retained for audit (minimum 5 years)
- [ ] Lessons learned documented
- [ ] Offboarding completion confirmed by: _______________ Date: ___________
